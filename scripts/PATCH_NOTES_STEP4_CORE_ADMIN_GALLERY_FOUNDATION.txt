What changed:
1. Rating schema alignment
   - Added Drizzle unique constraints for:
     - registry_entry_ratings(registry_entry_id, voter_profile_id)
     - partner_ratings(partner_id, voter_profile_id)
   - This aligns code schema with migration 0005_community_ratings.sql and protects the “one member = one vote, update later” logic.

2. Certificate / assessment consistency
   - When admin issues a USG certificate, the dog admin assessment is now automatically synced to:
     - registryDecision: usg_certified
     - certificateDecision: usg_certified
   - When admin revokes a certificate, certificateDecision is moved back to usg_candidate if it was usg_certified.
   - This keeps public profile, owner read-only view, Review queue, and Verify flow consistent.

3. Safer published Registry removal
   - Admin Registry “Remove” for already-published profiles is now a soft public removal:
     - registry entry becomes inactive
     - dog returns to approved/private
     - active certificate is revoked
     - admin assessment is moved away from active certified status
   - This avoids destructive deletion when the admin only wants to remove a profile from public Registry.

4. Cache / route revalidation
   - Added Gallery and certificate-code verify revalidation where needed.
   - Registry detail path is revalidated when a profile is removed from the public Registry.
